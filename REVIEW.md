## 🚨 关键风险（第四轮更新）

### 资金安全风险 (P0)
1. **竞态条件** - 并发请求可导致双重下注
2. **边池计算错误** - BettingRound.createSidePot实现完全错误
3. **金额参数无校验** - 可构造负数/超大金额攻击
4. **恢复后游戏规则错误** - 盲注为0，买入无限制
5. **JWT_SECRET可能未配置** - 可能接受alg:none令牌
6. **balance无数据库约束** - 并发时可能为负

### 前端安全风险 (P0)
1. **Token持久化到localStorage** - XSS可窃取所有token
2. **Socket无token刷新机制** - 游戏中途token过期静默断开
3. **断线重连不恢复订阅** - 重连后收不到游戏事件

### 认证安全风险 (P0)
1. **登录失败追踪用内存Map** - 多实例可绕过锁定
2. **Refresh Token存在Replay攻击** - 无原子删除
3. **Socket.io多实例不互通** - replicas:2但无Redis adapter
4. **数据库无密码** - PostgreSQL/Redis裸奔
5. **user.ts绕过中间件** - extractUserId与verifySignature冲突

### 功能正确性风险 (P1)
1. **isRoundComplete有Bug** - const重赋值编译报错
2. **庄家位置硬编码** - 不考虑实际玩家数量
3. **多设备登录互相覆盖** - 用户状态管理问题
4. **审计日志静默丢失** - 关键操作无记录
5. **gameState无schema校验** - 恶意数据可能导致崩溃
