## 核心模块

### 1. 游戏引擎 (GameEngine)

**职责**：管理德州扑克完整游戏流程

**关键特性**：
- 状态机管理牌局阶段
- Mutex 保护并发访问
- 手牌评估 (pokersolver)
- 快照保存与恢复

**位置**：`packages/poker-engine/src/game/GameEngine.ts`

```typescript
// 核心类
class GameEngine {
  // 线程安全：使用 Mutex
  private actionMutex: Mutex;
  
  // 处理玩家动作（async）
  async processAction(playerId, action, amount): Promise<ActionResult>
  
  // 获取游戏状态
  getState(): GameState
}
```

### 2. 手牌评估器 (HandEvaluator)

**职责**：评估牌型大小

**算法**：使用 pokersolver 库 (Cactus Kev)

**位置**：`packages/poker-engine/src/evaluator/HandEvaluator.ts`

### 3. WebSocket 服务 (Socket.io)

**职责**：实时双向通信

**特性**：
- 房间隔离
- 断线重连
- 状态同步

**位置**：`apps/server/src/services/socket.ts`

### 4. 认证服务 (Auth)

**职责**：用户身份验证

**机制**：
- JWT Access Token (短期)
- Refresh Token (长期，httpOnly cookie)
- Token 家族追踪 (防被盗用)

**位置**：`apps/server/src/routes/auth.ts`

## 关键技术决策

### 1. Node.js + TypeScript

**选择理由**：
- 团队熟悉度高
- 快速原型开发
- npm 生态丰富

**风险**：
- CPU 密集型任务可能阻塞 (但游戏引擎已优化)
- 单线程限制 (但 WebSocket 扩展性好)

### 2. Socket.io vs 原生 WebSocket

**选择理由**：
- 自动回退 (WebSocket → Long Polling)
- 房间抽象简化开发
- 成熟稳定

**备选**：原生 WebSocket (性能更好，但需要自己实现重连)

### 3. PostgreSQL vs MySQL

**选择理由**：
- JSONB 支持游戏配置
- MVCC 更好的并发控制
- 窗口函数支持复杂统计

### 4. Redis 多重角色

| 用途 | 数据结构 | 说明 |
|------|---------|------|
| 会话缓存 | String | JWT 黑名单 |
| 牌桌状态 | Hash | 热数据 |
| 限流控制 | String + Lua | 防刷 |
| 排行榜 | Sorted Set | 实时排名 |

### 5. Turborepo Monorepo

**优势**：
- 增量构建
- 任务缓存
- 依赖管理
